1. Six Assessment Groups

v3 uses 6 parallel LLM prompts to evaluate candidates, replacing the sequential 15-prompt v1 system.

1.1 Core Experience

Evaluates previous_role and responsibilities using a 6-level "Levels of Work" framework:

LevelDescriptionTime Window
Hands OnExecution-focused5 years
DiagnosisProblem-solving5 years
SystemsProcess design5 years
IntegrationCross-functional7 years
BusinessStrategic outcomes10 years
GlobalEcosystem-wide15 years

Overqualification gate: If delta between candidate LoW and job LoW >= +2, result = false

1.2 Skills & Knowledge

Requires POSITIVE evidence of something related. There must be a specific skill, project, or responsibility you can point to. Without such evidence, return false, not partially.

1.3 Experience Quantitative

  • Total years: Tolerant ranges (min × 0.9 to max × 1.3); flags over-experience (×2, ×3)
  • Context experience: Direct or strong indirect evidence
  • Industry fit: Narrow matching; consulting/vendor = implied evidence

1.4 People Management

Direct management only (not dotted-line). Team size: partially up to 2× smaller than required. Composition: explicit or inferred.

1.5 Education & Credentials

Critical v3 change: Missing education data = partially, not false. "No data ≠ absence of education."

Degree equivalencies: Master ↔ Specialist, PhD ↔ Candidate of Sciences, LLM ↔ Master

1.6 Logistics & Preferences

Languages — 5 implied evidence sources: CV language, work location, education, cultural markers, international company context.

Proficiency: 1 step below = partially; 2+ steps below = false.

Timezone — within range = true; 1-2h outside = partially; 3+h = false.

Must-Have Ratio

MH ratio = count(must_have where fit=true) / count(all must_have)

3. Decision Thresholds

MATCH

  • score ≥ 65 AND zero false must-haves
  • OR score ≥ 75 AND ≤1 false MH AND MH ratio ≥ 0.7

PARTIAL_MATCH

  • score ≥ 45 AND ≤2 false must-haves
  • OR score ≥ 35 AND zero false must-haves

NO_MATCH

  • All other cases
  • Also forced if base_score < 35 (responsibilities cap)

4. Responsibilities Cap (v3 Innovation)

Prevents generic responsibility descriptions from masking fundamental mismatches:

  1. Calculate base_score (score without responsibilities segment)
  2. If base_score < 35 → decision capped at NO_MATCH
  3. If any critical must-have = false → responsibilities cannot elevate to MATCH

5. Implementation (Python)

def calc_v3_score(assessment):
    fit_values = {'true': 1.0, 'partially': 0.5, 'false': 0.0}
    weights = {'must_have': 1.5, 'nice_to_have': 0.5}

    resp_items = [a for a in assessment if a['segment'] == 'responsibilities']
    other_items = [a for a in assessment if a['segment'] != 'responsibilities']

    base_weighted, base_max = 0, 0
    false_must_haves, must_have_true, must_have_total = 0, 0, 0

    for item in other_items:
        w = weights.get(item['requirement'], 1.0)
        val = fit_values.get(item['fit'], 0.0)
        base_weighted += val * w
        base_max += w
        if item['requirement'] == 'must_have':
            must_have_total += 1
            if item['fit'] == 'true': must_have_true += 1
            elif item['fit'] == 'false': false_must_haves += 1

    resp_weighted = sum(fit_values.get(r['fit'], 0) * 1.5 for r in resp_items)
    resp_max = len(resp_items) * 1.5

    score = ((base_weighted + resp_weighted) / (base_max + resp_max) * 100)
    base_score = (base_weighted / base_max * 100) if base_max else 0
    mh_ratio = must_have_true / must_have_total if must_have_total else 0

    # Decision
    verdict = 'NO_MATCH'
    if (score >= 65 and false_must_haves == 0) or \
       (score >= 75 and false_must_haves <= 1 and mh_ratio >= 0.7):
        verdict = 'MATCH'
    elif (score >= 45 and false_must_haves <= 2) or \
         (score >= 35 and false_must_haves == 0):
        verdict = 'PARTIAL_MATCH'

    # Responsibilities cap
    if base_score < 35:
        verdict = 'NO_MATCH'

    return {'score': round(score,1), 'base_score': round(base_score,1),
            'verdict': verdict, 'false_must_haves': false_must_haves,
            'must_have_ratio': round(mh_ratio, 2)}
