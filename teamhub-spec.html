1. Problem & Vision

Problem

Teams using Claude Code have no shared memory. Each developer starts every session from scratch. Team conventions, API standards, architecture decisions, deployment procedures — all of this lives in people's heads, scattered docs, or Notion pages that Claude never sees.

Result: Claude reinvents the wheel, writes code that violates team standards, and each team member spends time re-explaining the same context.

Vision

TeamHub — shared knowledge base that automatically injects relevant team context into Claude Code sessions. The developer writes a message, TeamHub finds the most relevant docs from the team's knowledge base, and prepends them to the message before Claude sees it.

Claude gets <teamhub_context> with team standards, API docs, architecture decisions — and writes code that fits the team from the first try.

Before / After

Without TeamHub

Developer: "Add a new API endpoint for users"

Claude: *writes Express-style route*
Developer: "No, we use Hono..."
Claude: *rewrites with Hono*
Developer: "And we always validate with Zod..."
Claude: *adds Zod*
Developer: "And our error format is..."
Claude: *changes error handling*

4 iterations to get basic endpoint right.

With TeamHub

Developer: "Add a new API endpoint for users"

TeamHub auto-injects:
  <teamhub_context hub="acme-backend">
  ## API Standards
  - Framework: Hono
  - Validation: Zod schemas in /schemas
  - Error format: { error: string, code: number }
  - Auth: Bearer token middleware
  </teamhub_context>

Claude: *writes correct Hono + Zod endpoint*

1 iteration. Done.

2. Core Concepts

ConceptDescription
Hub A directory with team knowledge files (markdown, text, code snippets). One hub per team/project. Lives on disk at a configured path (e.g. ~/team-hubs/acme-backend).
Document A single file in the hub. Typically markdown: api-standards.md, deployment.md, architecture.md. Can also be code files as examples.
Project mapping Links a working directory to a hub: "when I'm in /Users/dev/acme-api, use the acme-backend hub." Multiple projects can share one hub.
Context injection Automatic prepending of relevant hub documents to user messages before Claude sees them. Triggered by Claude Code hook or web UI reply.
Search Semantic search over hub documents. Given a user message, find the 3-5 most relevant docs. Uses BM25 (local) + optional Gemini reranking.

4. CLI Interface

Installation

npm install -g teamhub

Commands

CommandDescriptionStatus
teamhub init <path> --name <hub-name> Create a new hub directory with template docs and register in config P0
teamhub add <file> [-p <project>] Add a document to the hub. Auto-detects hub from current directory. P0
teamhub search <query> [-p <project>] [--top N] [--raw] Search hub for documents relevant to query. Returns formatted context block. P0
teamhub inject [-p <project>] [--max-tokens N] Return full hub context (all docs, truncated to token budget). Fallback when search returns nothing. P0
teamhub link <project-path> <hub-name> Map a project directory to a hub in config. P0
teamhub list List all hubs and their project mappings. P1
teamhub index [hub-name] Rebuild BM25 index for a hub (or all hubs). Automatic on first search. P1
teamhub hook install Register TeamHub as a Claude Code hook in ~/.claude/settings.json. P0
teamhub hook uninstall Remove the Claude Code hook. P1
teamhub docs [hub-name] List all documents in a hub with token counts. P1
teamhub remove <file> Remove a document from the hub and re-index. P2

Examples

# Create a hub for backend team
teamhub init ~/team-hubs/acme-backend --name acme-backend

# Add documentation
teamhub add api-standards.md
teamhub add deployment-guide.md
teamhub add error-handling.md

# Link projects to hub
teamhub link ~/code/acme-api acme-backend
teamhub link ~/code/acme-workers acme-backend

# Search (used by hook/web UI automatically)
teamhub search "add new API endpoint" -p ~/code/acme-api --top 3

# Install Claude Code hook
teamhub hook install

# Now every Claude Code session in ~/code/acme-api
# automatically gets relevant context from acme-backend hub

5. Configuration

Config file: ~/.teamhub/config.yaml

hubs:
  acme-backend:
    path: ~/team-hubs/acme-backend
    team:
      - alice@acme.com
      - bob@acme.com
    search:
      bm25_keep_ratio: 0.3       # BM25 filtering: keep top 30% of docs
      gemini_input_tokens: 100000 # Token budget for Gemini reranking input
      gemini_output_tokens: 800   # Token budget for Gemini reranking output
      top_k: 3                    # Return top 3 docs

  acme-frontend:
    path: ~/team-hubs/acme-frontend
    team:
      - alice@acme.com
      - charlie@acme.com
    search:
      bm25_keep_ratio: 0.3
      top_k: 5

projects:
  ~/code/acme-api: acme-backend
  ~/code/acme-workers: acme-backend
  ~/code/acme-web: acme-frontend
  ~/code/acme-mobile: acme-frontend

Search settings explained

SettingRangeDefaultWhat it does
bm25_keep_ratio0.05 - 1.00.3After BM25 scoring, keep top N% of documents. Lower = stricter filtering, faster. Higher = more context, risk of noise.
gemini_input_tokens10K - 500K100KMax input tokens for Gemini reranking. 0 = skip Gemini (BM25 only).
gemini_output_tokens500 - 8K800Max output tokens from Gemini reranking.
top_k1 - 103Number of documents to return in context block.

6. Claude Code Hook Integration

This is the key feature

TeamHub works as a Claude Code hook — it automatically injects context into every session without any action from the developer. No web UI needed. Works in terminal. Zero friction.

Hook registration

teamhub hook install adds this to ~/.claude/settings.json:

{
  "hooks": {
    "UserPromptSubmit": [
      {
        "matcher": "",
        "command": "teamhub hook-handler --event UserPromptSubmit"
      }
    ]
  }
}

How the hook works

Event: UserPromptSubmit
  │
  ▼
teamhub hook-handler receives:
  - session_id
  - user message text
  - cwd (project directory)
  │
  ▼
1. getHubForProject(cwd) → hub name
   If no hub → exit (no context needed)
  │
  ▼
2. messageNeedsContext(message)?
   Skip trivial messages: "ok", "continue", "y", short commands
   If no → exit
  │
  ▼
3. teamhub search "{message}" -p {cwd} --top {top_k} --raw
   BM25 + optional Gemini rerank
  │
  ▼
4. Output context block to stdout:
   <teamhub_context hub="acme-backend">
   ## API Standards
   - Framework: Hono
   ...
   </teamhub_context>
  │
  ▼
Claude sees: context block + original message

messageNeedsContext logic

Not every message needs context injection. Skip to avoid wasting tokens:

Message typeInject?Example
Trivially short (<10 chars)No"ok", "y", "done"
Continuation patternsNo"continue", "go ahead", "keep going", "lgtm"
Short without question or codeNo"sounds good", "nice work"
Substantive questionYes"Add a new endpoint for user profiles"
Code referenceYes"Fix the UserService.getById method"
Question with ?Yes"How do we handle auth in this project?"

Multiple hooks can coexist

TeamHub + Context Trash = best combo

Claude Code supports multiple hooks on the same event. TeamHub (inject context) and Context Trash (compress tool results) work together:

{
  "hooks": {
    "UserPromptSubmit": [
      {
        "matcher": "",
        "command": "teamhub hook-handler --event UserPromptSubmit"
      }
    ],
    "PostToolUse": [
      {
        "matcher": "",
        "command": "context-trash-hook --event PostToolUse"
      }
    ]
  }
}

TeamHub adds relevant context IN. Context Trash compresses bloated results OUT. Together they maximize useful context density.

7. Search Pipeline

Two-stage search

Stage 1: BM25 (local, free, fast)
  ├── Tokenize query into terms
  ├── Score each document in hub
  ├── Keep top bm25_keep_ratio (default 30%)
  └── Return ranked docs
       │
       ▼
Stage 2: Gemini rerank (optional, ~$0.001)
  ├── Send query + BM25 candidates to Gemini Flash
  ├── "Which of these documents are most relevant to: {query}"
  ├── Gemini returns ranked list with relevance scores
  └── Return top_k documents

BM25 indexing

  • Index stored at ~/.teamhub/index/{hub-name}.json
  • Auto-rebuilt when hub files change (mtime check)
  • Fields: document path, title (first H1), body text, tokens
  • Implementation: lightweight BM25 in ~100 lines of TypeScript, no external dependencies

When Gemini is skipped

  • gemini_input_tokens: 0 in config — BM25 only mode
  • No Gemini API key configured
  • Hub has ≤5 documents (BM25 is sufficient)
  • Gemini API call fails — graceful fallback to BM25 results

Context block format

<teamhub_context hub="acme-backend" docs="3" tokens="~2400">

## From: api-standards.md (relevance: 0.92)
- Framework: Hono v4
- All routes in /src/routes/, grouped by resource
- Validation: Zod schemas in /src/schemas/
- Error format: { error: string, code: string, details?: any }
- Auth: Bearer token via middleware, never inline

## From: database.md (relevance: 0.78)
- ORM: Drizzle
- Migrations in /drizzle/
- Always use transactions for multi-table writes

## From: naming.md (relevance: 0.65)
- Files: kebab-case
- Variables: camelCase
- DB columns: snake_case

</teamhub_context>

8. Web UI (Claude Session Manager)

Already implemented

Web UI integration in Claude Session Manager is already working:

  • Settings page — enable/disable TeamHub, configure search params per hub (BM25 ratio, Gemini tokens, top_k)
  • Reply handler — /api/sessions/[id]/reply calls teamhub search before sending message
  • Action log — every injection logged with hub name, token estimate, context content
  • Health check — /api/health reports TeamHub availability

10. Implementation Phases

Phase 1: Core CLI P0 — MVP

  • teamhub init — create hub directory with template docs
  • teamhub add — copy file into hub
  • teamhub link — map project → hub in config
  • teamhub inject — return all docs (truncated to max tokens)
  • Config read/write (~/.teamhub/config.yaml)

Test: teamhub init ~/test-hub --name test && teamhub inject -p . returns docs.

Phase 2: BM25 Search P0

  • BM25 implementation (tokenize, score, rank)
  • Auto-index on first search (store at ~/.teamhub/index/)
  • Re-index when files change (mtime check)
  • teamhub search "query" — returns top-K relevant docs

Test: Hub with 10 docs. Search "deployment" returns deployment.md, not api-standards.md.

Phase 3: Claude Code Hook P0 — Key UX

  • teamhub hook install — register in ~/.claude/settings.json
  • teamhub hook-handler — receive event, search, output context
  • messageNeedsContext() — skip trivial messages
  • Verbose logging to stderr (hub name, docs found, tokens)

Test: Start Claude Code in linked project directory. Type a question. Verify context was injected (check ~/.claude/projects/*/actions.log or verbose output).

Phase 4: Gemini Reranking P1

  • Optional Gemini Flash integration for semantic reranking
  • Falls back to BM25-only if no API key or Gemini fails
  • Config: gemini_input_tokens, gemini_output_tokens
  • API key stored in ~/.teamhub/config.yaml or env var GEMINI_API_KEY

Phase 5: Polish & Publish P1

  • teamhub list, teamhub docs, teamhub remove
  • README with examples and screenshots
  • teamhub hook uninstall
  • npm publish as teamhub

Phase 6 (Future): Team Sync P2

  • Hub backed by git repo — teamhub sync pulls latest
  • Team members can push updates to shared hub
  • Conflict resolution for concurrent edits
  • Hub versioning (tag releases of team knowledge)

12. Risks & Mitigations

RiskImpactMitigation
Context injection wastes tokens on irrelevant docs High — pushes toward compact faster Strict messageNeedsContext filtering. Configurable top_k and bm25_keep_ratio. User can disable per-project.
BM25 returns wrong docs for ambiguous queries Medium — Claude gets misleading context Gemini reranking as second stage. Relevance score threshold (don't inject low-relevance docs). User can test search before relying on it.
Hook latency slows down every message Medium — bad UX BM25 is local and fast (<50ms). Gemini adds 1-2s but is optional. Hook timeout: 5 seconds max, then skip.
YAML parser breaks on edge cases Low — config corruption Minimal parser only handles known structure. Test with all real-world configs. Backup config before write.
npm name teamhub taken Medium — need rename Check npm registry. Fallbacks: @teamhub/cli, teamhub-ai, teamhub-claude.
Hub docs get stale / outdated Medium — Claude follows old standards Phase 6 git sync. teamhub docs shows last-modified dates. README warns about maintenance.
Conflict with Context Trash hook Low — hooks are independent Different events: TeamHub on UserPromptSubmit, Context Trash on PostToolUse. No conflict.
